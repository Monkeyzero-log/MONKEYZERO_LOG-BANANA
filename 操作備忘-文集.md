# 操作備忘（文集）

維護文集會用到的步驟。與習作輯的備忘是同一套邏輯，指令完全相同。

---

## 一、日常上傳

改完任何東西，在 VS Code 按 <code>Ctrl + `</code> 打開終端機：

```
git add .
git commit -m "這次改了什麼"
git push
```

commit 訊息例：`文集 卷一新增：篇名`、`修訂 發刊詞 錯字`

---

## 二、新增一篇文章

### 1. 建立篇的頁面

- 複製 `pieces/_template-piece.html`，改名（**全英文加連字號**，例：`pieces/2026-09_night-rain.html`）
- 打開新檔案，照裡面的註解修改：
  - `<title>`、篇號、篇名、日期、字數
  - 最上方 `crumbs` 和最下方 `pager` 的返回連結，改成所屬的卷：
    原創 `../original.html`／同人 `../fanwork.html`／研究 `../research.html`
  - **原創與研究篇**：把整個 `<div class="fan-note"> ... </div>` 區塊刪掉
  - **同人篇**：保留聲明欄，原作、角色、分級、聲明照實填

### 2. 貼上正文

- 一段文字一個 `<p>`，會自動縮排兩字：

```html
<p>第一段。</p>
<p>第二段。</p>
```

- 分節的地方插入：

```html
<div class="brk">＊</div>
```

- 篇末保留：

```html
<p class="fin-line">（完）</p>
```

- 研究筆記的小標用：

```html
<h3>觀察</h3>
```

- 某段不想縮排（例如引文後的接續段）：`<p class="noindent">`

### 3. 到所屬的卷加一列篇目

打開 `original.html`（或 fanwork／research），複製一列 `<a class="toc-row">`，改連結、篇號、篇名、日期和字數。

### 4. 上傳

回到「一、日常上傳」三行指令。

## 三、字數統計

寫完把全文貼到任何線上字數統計工具數一次（或在 VS Code 裝擴充功能 chars-counter），手動填進兩個位置：篇目那一列、篇頭的 `piece-meta`。格式：`計 3,240 字`。

## 五、常見問題

**點進某一篇變成白底沒樣式**
`pieces/` 裡的檔案讀樣式走的是 `../style.css`（上一層）。如果你把篇的檔案移出 pieces 資料夾、或複製模板時放錯位置，路徑就斷了。篇的頁面一律放在 `pieces/` 裡。

**篇目點下去 404**
連結和檔名沒對上，九成是大小寫或多打少打一個字。GitHub 把 `Night-Rain.html` 和 `night-rain.html` 當兩個不同的檔。

**改了沒生效**
等一兩分鐘讓 Pages 部署，再 Ctrl + F5 強制刷新。

**想改整本冊子的外觀**
所有頁面共用 `style.css`，顏色都集中在最上面的 `:root` 區塊，改一處全書生效。
