# 第二章：引擎已知問題（候選 v12.2）🔴

## 2.1 「CPU過載，降級渲染...」永遠誤報

**根因**：24.1 PerformanceMonitor 量的是「幀與幀的間隔」（delta），
但 requestAnimationFrame 被螢幕更新率鎖死——60Hz 螢幕上 delta 恆為 ~16.7ms，
`cpuUsage = 16.7 / 16.67 × 100 ≈ 100 > 80` → 每秒警告一次，跟真實負載無關。

猴子解釋：警衛用「你多久進出一次大門」判斷你累不累，
但大門一分鐘只開 60 次，所以每個人看起來都累到不行。

**佐證**：console 看 `ENGINE.perfMonitor.fps` 是漂亮的 ~60（明明不卡）；
120Hz 螢幕上同一份程式碼警告反而消失（delta ~8.3ms → cpuUsage ~50）。

**實際影響**：零。degradeComplexity 是 🔴 未實作的預留鉤子（optional chaining 靜默跳過），
只洗版 console，不影響任何 demo 結果。

**修法（量「這一幀實際工時」而非幀間隔）**：

```js
// animate() 開頭：
const _frameStart = performance.now();
this.perfMonitor.update();

// animate() 結尾（requestAnimationFrame 那行之前）：
this.perfMonitor.recordWork(performance.now() - _frameStart);

// PerformanceMonitor 新增：
recordWork(workMs) {
  this.cpuUsage = (workMs / 16.67) * 100;   // 實際工時 ÷ 幀預算
  if (this.frameCount % 60 === 0 && this.cpuUsage > 80) {
    console.warn('CPU過載，降級渲染...');
    if (typeof ENGINE !== 'undefined') ENGINE.degradeComplexity?.();
  }
}
// 並把 update() 裡原本的 cpuUsage 計算與警告區塊移除
```

猴子解釋：以前量「你多久回來一次」，現在量「你回來之前真的工作了多久」。
空閒等待 vsync 的時間不再被算成加班。

## 2.2 同根源的隱患：120Hz 宇宙 2 倍速 🔴

`LIGHT_SPEED: 60` 與所有速度都以「每幀」為單位。120Hz 螢幕上 rAF 跑 120fps，
整個宇宙用 2 倍速運行。根治要做 deltaTime 校正，屬 v13 等級架構決定，先記著。

# 第三章：語言與文字物理學

引擎骨子裡是為英文設計的，中文會撞上這些（都對應具體程式碼）：

| # | 機制 | 對中文的影響 | 對策 |
| - | ---- | ------------ | ---- |
| 1 | computeSemanticVector 只統計 A–Z | 中文向量全零 → 共鳴系統全滅（相似度恆 0）| 🔴 換指紋算法（Unicode 區段雜湊或按部首）|
| 2 | 質量 = 字符數 | STONE(5字符) 比 石頭(2字符) 重 2.5 倍，語義顛倒 | ✅ 手動設 semanticWeight（每漢字約 2.5）|
| 3 | 相變 fallback 用 toLowerCase/toUpperCase | 中文無大小寫，靜靜失效 | ✅ 本來就該手動設三態文字——中文反而能玩**換字**：冰→水→汽 化 中、磐石→泥濘→塵 埃，表現力比大小寫強 |
| 4 | getAdjacentKey 是 QWERTY 鄰接表 | 中文變異大多無效；插入型會塞英文字母進中文詞 | 🎨 自建**形近字突變表**：土→士、未→末、日→曰、人→入、天→夭。「明天見」進化三代變「明夭見」，比 STONE→STOQE 毛骨悚然 |
| 5 | 情緒向量場的 pos/neg 是英文詞表 | 中文台詞命中不了，向量場是死的 | 🎨 換中文詞表（愛恨怕死光暗⋯）|
| 6 | 排版 | CJK 等寬方塊字：AABB 更規則、letter-spacing 效果更均勻——**中文的主場** | 加分項 |

**概念級彩蛋：直書 = 重力轉 90 度。** 公理三說「重力 = 換行」，
`writing-mode: vertical-rl` 時換行方向變成水平——恐怖章節切直書，
整個宇宙的重力橫著走。英文做不到。

# 第四章：符號物性表 🎨

Codex 1.3 早有伏筆：Level 0 消失是「變成句點」——標點本來就是這個世界觀的物質。

| 符號 | 物性 | 用法 |
| ---- | ---- | ---- |
| 。、 | 水滴 / 塵埃 | 台詞消散時碎成標點落下 |
| ⋯⋯ | 煙、遲疑 | 哽咽台詞尾端裊裊上飄；停頓時間比普通字長 |
| ～ | 液體 / 波 | 醉話、水面、暈眩的基線 |
| ／＼｜＿ | 地形 | 直接用字符砌斜坡、牆、地板 |
| ─ ╱ ╲ | 心電圖線 | 見 6.3；死亡 = 振幅歸零變 ──────── |
| ！ | 衝擊粒子 | 爆點時從字裡炸出 |
| 全形 vs 半形 | 重 vs 輕 | 「！」比「!」重，字面意義上的 |
| 注音文 | 氣音 / 童聲 | 「ㄅ、ㄅ要過來」——zh-TW 限定的耳語 |
| 組合附加符（Zalgo）| 侵蝕 / 瘋狂 | 台詞被 c̷o̶r̷r̸u̸p̶t̷ 侵蝕，中文也可疊 |

# 第五章：遊戲架構——三層 ✅（核心決策）

「台詞固定、偶爾演出有物理狀態」→ 需要的不是常駐世界模擬器，而是**演員**。

| 層 | 職責 | 技術 | 使用頻率 |
| -- | ---- | ---- | -------- |
| 1. 對話層 | 對話框、固定台詞 | 普通 HTML | 常駐 |
| 2. 演出層 | 逐字級情緒效果（抖動/呼吸/心電圖⋯）| 逐字 span + CSS/JS | 遊戲主力 |
| 3. 引擎層 | 大場面：詞掉出對話框摔碎、蒸發、爆炸 | TextualPhysicsEngine | 省著用，演完釋放 |

**關鍵理由**：心顫/呼吸/心電圖是**逐字級**效果，而引擎把整段文字當一顆剛體
（一個 span），只能整句移動——根本抖不出「每個字各自顫」。
這些正是 Codex 標 🎨 的領域，手寫 CSS/JS 的地盤。
好處：輕量、可控、台詞不會被天擇改成錯字。

# 第六章：演出目錄（POC 1 已實作）✅

## 6.1 核心機制
- `wrapChars()`：台詞拆成逐字 `<span class="ch">`
- 所有效果共用 `cleanup()`：切換不殘留
- `prefers-reduced-motion` 開啟時全部自動停用（暈眩敏感玩家友善）
- 接遊戲的方式：台詞資料加一欄 `{ speaker, text, fx: 'tremble' }`

## 6.2 六種效果與實作要點

| 效果 | 要點 |
| ---- | ---- |
| 緊張顫抖 | 每個字的振幅、頻率**各自隨機**（CSS 變數 --jx/--jy/--jd），才像真的發抖而不是整句做操 |
| 呼吸 | 整句 scale + 字距 + 透明度的正弦循環，約 4 秒一息 |
| 心電圖 | 脈衝掃過逐字：R 波頂起 + S 波回落，字色閃紅；下方用 ─╱╲ 字符畫同步基線。**死亡演出 = 振幅歸零，基線拉平**，一句程式碼 |
| 寒冷 | 字距收縮 -1px、色偏藍白、細顫 + 週期性整句大寒顫 |
| 哽咽 | 逐字浮現；「⋯」停頓 ×5、「，」×3——**標點的質量直接變成時間節奏**；句尾字下沉變暗 |
| 怒吼 | 放大加粗猛震；驚嘆號比其他字抖得更兇（.bang class）|

## 6.3 演出層的物理直覺
標點有物理（停頓 = 質量）、每字有個體差（隨機參數）、
狀態有生命週期（cleanup）——跟引擎同一套世界觀，只是不跑模擬。

# 第七章：地圖・行走・轉場

## 7.1 已做進 POC 2 ✅

| 機制 | 內容 | 世界觀對應 |
| ---- | ---- | ---------- |
| 路就是字 | 地面逐字鋪成：草原＿、淺灘～、雪地‥ | 「距離 = 字數」，公理三的字面實現 |
| 地形物性 | 水中速度 ×0.5 + 浮沉；查表即可，不需碰撞系統 | — |
| 腳步聲是字 | 草「沙」、水「啪嚓」、雪「吱」，冒出上飄淡去 | 聲音不用音效檔，用字演 |
| 足跡是標點 | 身後留「、」漸淡 | 遺忘之點的世界觀 |
| 迷霧 = 亂碼 | 沒去過的地名跳動亂碼，走近 250px 逐字解碼定形；迷你地圖同步 | 「存在 = 可被解析」，公理一 |
| 崩塌轉場 | 畫面上**每一個字符**（地面/地名/足跡/你）各自帶隨機延遲角速度受重力墜落散開，落盡後新場景以字距展開浮現 | Level 2 消失的演出；引擎級大場面的簡化版 |

## 7.2 未做構想 🎨

- **縮放 = 標題層級**：Codex Z 軸表直接用——h1 國家、h2 城鎮、p 街道；「放大地圖」= 往語義深處鑽一層。
- **快速旅行 = 蟲洞**：Codex 第八章，去過的地名變 `<a>` 錨點，O(1) 瞬移，零成本。
- **地圖即散文**：地圖是一段描述文字，地名嵌在句中發光；旅行 = 游標在文章裡移動。
- **睡眠/昏迷轉場 = Level 0**：全畫面文字收縮聚攏成一顆「.」，停一拍，再從句點展開新的一天。（Codex 1.3：語義消失 = Z→∞ 變句點。**建議當遊戲招牌轉場**。）
- **蒸發轉場**：舊場景文字相變離場——字距拉開、變淡、上飄成「S T E A M」狀。適合夢醒、回憶結束。
- **直書轉場**：回憶/靈異段落切 writing-mode，重力軸轉 90 度。
- **墨滲轉場**：新場景從模糊墨點滲開變清晰（blur 收斂）。適合讀信、日記。
- **負重感**：背包物品是有質量的詞；拿太多重物，「你」走路下沉更深、腳步聲變「咚」。

# 第八章：加碼構想（對文字遊戲殺傷力最大的兩個）🎨

## 8.1 選項的物理
- 說謊的選項微微顫抖
- 不敢說出口的選項慢慢下沉、變淡、最後掉出選單（**時間壓力 = 重力**）
- 游標懸停的選項開始呼吸
- 玩家還沒選，身體語言已經先演完了

## 8.2 狀態不用血條
- 理智值下降 = 台詞逐漸被組合附加符侵蝕（Zalgo 爬上文字）
- 體力下降 = 「健康」這個詞自己掉筆畫、掉字
- UI 本身就是敘事

## 8.3 其他
- 天氣層：雨 = 「、」落下、雪 = 「。」慢落、風 = 全畫面字距漂移
- 回憶：模糊 + 變暗；被遺忘的記憶留下可 hover 召回的小點

# 第九章：實作心得（POC 過程學到的）

1. **旁白跟著座標走**：POC 2 的 `NARRATIONS` 表（x 座標 → 該浮現的句子）
   其實就是敘事系統雛形——地圖、事件、台詞可全部收斂成
   「座標 → 內容」的資料表，程式碼不用再長。**建議正式版沿用此模式。**
2. **resize 時地面跟著搬**：直接跟引擎 v12.0 的 groundY 修正學的，同一個坑不踩兩次。
3. **效果一定要有 cleanup**：rAF、setTimeout、className 統一收，切換才不殘留。
4. **每字隨機參數**是「像活的」的關鍵；整齊劃一就變做操。
5. **對照組是驗收的一部分**：不被影響的物體（SAFE/DARK/SILENT）證明衰減與範圍正確。
