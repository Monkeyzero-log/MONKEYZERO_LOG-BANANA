# 第一章：逐字演出機制 ✅

## 1.1 逐字包裝（一切的地基）
- **出處**：POC 1 起全系列
- **做法**：`wrapChars(text)` 把台詞拆成 `<span class="ch">`，
  `display:inline-block` 才能各自 transform。
- **紀律**：每個效果都要有 `cleanup()`（rAF、setTimeout、className 統一收），
  切換不殘留。

## 1.2 情緒演出六式（POC 1）

| 效果 | 實作要點 |
| ---- | -------- |
| 緊張顫抖 | 每字隨機 CSS 變數 `--jx/--jy`（±0.8~1.6px）、`--jd`（0.08~0.16s）——**逐字隨機是「像活的」的關鍵**，整齊劃一就變做操 |
| 呼吸 | rAF 正弦：scale(1±0.025) + letter-spacing(1±1.5px) + opacity(0.86±0.14)，約 4 秒一息 |
| 心電圖 | 脈衝掃過逐字：R 波 `-22·e^(-d²/0.35)` 上衝 + S 波回落；經過字色閃紅；下方 `─╱╲` 字符基線同步。**死亡 = 振幅歸零基線拉平，一行程式** |
| 寒冷 | letter-spacing -1px + 色偏藍白 + 高頻小顫 + 週期整句大寒顫（keyframe 0/88/100% 靜、90~96% 抖） |
| 哽咽 | 逐字浮現；停頓 =「標點的質量」：`⋯`×450ms、`，`×320ms、普通字 90~150ms；句尾 2-3 字 `.droop`（下沉 4px + 變暗） |
| 怒吼 | 整句 scale/weight 上調 + 0.08s 猛震；`！`加 `.bang` class 抖更兇 |

## 1.3 標點＝樂譜 ✅
- **出處**：三框 mockup（沿用至全系列打字機）
- **做法**：`pauseAfter(ch)` 查表 `{，:170, 、:120, 。:330, ？/！:300, ⋯:260, —:180}`；
  機械聲部（mech:true）壓平為 120ms——**冷血的角色連標點都不呼吸**。

## 1.4 聲音的字體學 ✅
- **出處**：圓美、三框
- **做法**：每種聲部一個 class——私語（0.72em、淡紫 `#9b8fb0`、字距+2px）、
  念白（`#d9c07a`）、呼喊（w800 + 重陰影 `4px 5px 7px`）、
  細屑（0.66em 灰綠）、心聲（斜體 `#b9a3e0`）、系統（等寬字 mono）。
- **殺手級用法——樣式即敘事**：結尾主角自語的引文
  **重用**開頭外部批評者的 class → 聲音內化，零說明成本（圓美的題眼）。
- 聲部簽名還含**語速**（protag 52ms/字、other 74、sys 16、narr 38）
  與**是否呼吸**（.breathe 只給活人）。

## 1.5 自我示範的字 ✅
- **出處**：圓美
- **做法**：字義=字形——「重疊」`text-shadow:0.14em 0`、「太濃」w900、
  「太淡」opacity .3、「太尖」skewX(-14deg)、「太寬」ls .55em、「很窄」ls -0.12em。
- **價值**：零學習成本的特效（意義與外觀同一件事），最安全的一類。

## 1.6 同句異演 ✅
- **出處**：圓美（「大膽而堅定」→「顫抖而猶豫」）
- **做法**：重複句用不同 charFx 參數重演（amp 0→1.5px），
  讓讀者的眼睛自己比出差異。輪迴題材的核心技（找碴玩法）。

## 1.7 量子字 ✅
- **出處**：三框 mockup `{q:慈愛|馴化}`
- **做法**：span 每 700ms 切換兩個詞；預先量測最寬值鎖 min-width（不跳版）；
  hover 暫停（觀測坍縮）、touch 切換暫停。

# 第二章：空間・移動・鏡頭 ✅

## 2.1 地形＝字（距離＝字數）
- **出處**：POC 2
- **做法**：地面逐字鋪（`＿`草/`～`水/`‥`雪，CHAR_W=20px）；
  TERRAIN 段落表 `{ch, step聲, speed倍率, bob浮沉}`——
  **地形物性查表即可，不需碰撞系統**（水中 speed×0.5 + 浮沉 5px）。

## 2.2 腳步聲字＋足跡
- **做法**：每移動 ~90px 冒地形狀聲字（沙/啪嚓/吱）上飄淡出 0.95s；
  每 ~55px 留「、」足跡淡出 2.3s。聲音不用音效檔，用字演。

## 2.3 迷霧＝亂碼解碼
- **出處**：POC 2、輪迴
- **做法**：未知文字每 120~150ms 從 NOISE 池（骸霧咒祀…）換字；
  解鎖時逐字翻牌定形（每字 flips=4+i×3 次、70ms/次）。
  「存在＝可被解析」公理的互動化。

## 2.4 鏡頭系統（鏡頭＝敘事者）
- **出處**：符號舞台
- **做法**：世界一個 div，相機 `{x,y,z}` 每幀 lerp（ease 0.03~0.05），
  `transform: translate(vw/2 − x·z, vh·0.42 − y·z) scale(z)`。
- **鏡位表**：全景 z0.8 / 中景 1.5 / 近鏡 3.4 / **超近鏡 7.5**（ease 放慢 0.03）
  / 跟拍（`cam.follow=actor`，每幀 tx=演員 x）。
- **超近鏡的意義**：1px 的顫抖被放大成地震——微小演技升格為主戲。

## 2.5 對峙位（寶可夢站位）
- **做法**：甲移至 `camX−200`、scale 1.9、貼地；乙移至 `camX+210`、
  scale 1.05、抬高 76px；腳下各一個 CSS 橢圓光台淡入；鏡 z1.35。
  **不需要戰鬥系統，站位本身完成「對峙」敘事**；`endStandoff()` 全員歸位。

## 2.6 演技動詞表
- **出處**：符號舞台（actor FSM：idle/walk/bounce/down/kneel/sit）

| 動詞 | 要點 |
| ---- | ---- |
| walk(to, speed) | 回傳 Promise（可 await）；bob=\|sin\|×4；**速度即演技**（0.55=重傷蹣跚, 3=狂奔）；隨機掉足跡 |
| tremble | glyph 加 jitter class；配超近鏡食用 |
| collapse | glyph rotate(84°) 0.55s + opacity .45；噴 14 顆紅「、」（重力 0.22 落地留存）＋**筆畫碎片「丶丿一乚」**——符號碎成自己的筆畫 |
| kneel / situp | scaleY 0.72 壓低 / 恢復＋微傾 |
| bounce | 擠壓伸展：`scaleY(1−|cos|·0.14)`、`scaleX(2−scaleY)`＋頂點掉 ♪；cycles 參數可定次數（await） |
| idle | 呼吸 bob `sin(t/700+相位)×1.5`，相位逐人隨機 |

## 2.7 世界粒子與劇情連動
- **做法**：飄雪 = interval 生成 + `snowing` 旗標；
  **關鍵拍直接改旗標**（信使倒下→雪停）。讀者未必意識到，但一定感覺得到。

## 2.8 崩塌轉場（場景死亡）
- **出處**：POC 2
- **做法**：畫面每個字符變 body：隨機延遲 0~0.5s、vy 重力 0.55、角速度 ±14，
  墜出畫面後新場景以字距展開浮現。Level 2 消失的演出版。

# 第三章：框的戲劇學 ✅

> 「位置即語言」的下一句：「布局即劇情」。框的數量本身是台詞。

## 3.1 框事件動詞表（出處：三框戲劇學）

| 動詞 | 實作 | 敘事意義 |
| ---- | ---- | -------- |
| open（誕生） | 移除 .closed，max-height 過渡 0.7s | 角色第一次開口，框才存在 |
| close（失語） | .closed：max-height 0 + padding/border 歸零 | 說不出話＝畫面上沒有他的位置 |
| die（死亡） | .dying：translateY(120vh) rotate(5°) 1.25s → 靜默轉 closed | 載著那道聲音的框墜出畫面 |
| slot（遺缺） | .slot：壓成 22px 虛線空槽，標籤改「（空）」 | 空位一直空著——「沒人跟他說晚安」 |
| ghost（幽靈態） | .ghost：opacity .42 + 虛線邊 + 斜體 | 畫像、亡靈、錄音 |
| split（分裂） | 框內切換雙欄 splitbox，各自 who/txt | 雙面間諜的兩張臉，同拍分別發言 |
| merge（合併） | 收回單欄＋tremor | 兩張臉收回一張，但框在抖 |
| swap（易位） | **FLIP**：量舊位→換 DOM 序→反向補償→1.1s 歸位 | 「上·主角」的牌子沉到下面——他成了別人故事裡的聲部 |
| dim / undim（靜場） | 全框 opacity .22，只剩心電圖在走（BPM 降 30） | 無言的拍子 |

## 3.2 指揮棒與布局讀數
- conduct()：活躍框亮（opacity 1）其餘暗（.5）——視線引導。
- 右上角布局讀數「框：上·中·下(魂)」——設計研究儀表，正式版可拆。

## 3.3 心電圖分隔線
- SVG polyline，BPM 由每拍的 bpm 欄位 lerp（×0.02/幀）；
  QRS 波形手寫（q<0.06 R 波 −9、<0.12 S 波 +3、0.3~0.42 T 波 −2.2）。
  框與框之間的「活的邊界」。

## 3.4 紀律
框變化只在**拍與拍之間**；一拍最多一個布局事件；標籤跟著框走；
分裂/易位整場最多一兩次，留給轉折點。

# 第四章：消失・轉場・倒帶 ✅

## 4.1 台詞的六種死法（POC 3，統一介面 `fx(chars)`）

| 死法 | 要點 | 世界觀對應 |
| ---- | ---- | ---------- |
| 碎落 | 逐字（45ms 錯開）變「、/。」→ 重力墜落旋轉 | 台詞碎成標點（符號物性表） |
| 蒸發 | 逐字（70ms）transition 1.6s：上飄 50~80px + 散開 + 變亮變淡 | 相變 |
| 燃燒 | 火從左掃右（110ms/字）：燒紅 scale1.12 → 300ms 後化「﹒」灰點搖曳飄落 | — |
| 溶解 | 隨機順序：字→`～`(藍)→450ms→`＿`(灰)→450ms→淡出 | **Codex 5.2 的字面實現** |
| 句點化 | 全字 0.9s 塌縮到行中心 scale0.08 → 換一顆「.」→ 熄滅 | **Codex 1.3 Level 0**（推薦當「最後一句話」招牌） |
| 黑洞吞噬 | 逐字極座標旋入：`r=r0(1−p²)+14p²`、`θ=θ0+7p²`；紅移兩段 lerpColor（原色→#ff5544→#3a0e0a）；義大利麵化 stretch 1+1.6p、shrink 1−0.9p | **Codex 13.3 演出版** |

## 4.2 符號黑洞（道具）
- 核心「●」近黑 + 暖光暈；光子環正圓白熱；三層 `∙◦·` 字環
  反向差速旋轉、壓扁 squash 0.5 成吸積盤；
  塵埃每 0.8s 外緣生成螺旋墜入（越近越快：`r−=0.9+(130−r)·0.012`）。

## 4.3 彈射成缺口（圓美 v2 核心）
- 詩句逐字走上圓周（anglePer=2π/總字數；行走 hop `sin(tπ)×34`）；
- 關鍵句「距離完美還差一筆」整段彈射：留 **0.12 透明度殘影**（遺忘之點）、
  本體變拋體（徑向初速 3.5~5.5 + 重力 0.5）、**落地成殘骸躺著不動**（opacity .55）；
- 第二輪重演 → 地上並排**兩具同樣的句子**＝反覆循環寫在地板上。
- **擦除 vs 彈射的區別**（重要語義）：擦除（erased）邊落邊 alpha−0.03 化為烏有；
  彈射（debris）落地留存。被抹去的消失、被拋棄的留下屍體。

## 4.4 倒帶轉場（輪迴樣本招牌）
- 四件事並行：台詞**逐字倒打收回**（14ms/字）、
  碎片記住原點 `transition 1s` 飛回並淡出、
  時鐘倒撥（00:00→23:59→…→23:47 序列 110ms/格）、
  全屏 `filter:contrast(1.1) saturate(.6)` + 「◀◀ REWIND」閃爍。
- **倒帶本身就是規則說明書**——讀者看一次就懂輪迴，零旁白。

## 4.5 批評的物理攻擊（圓美 v2 動詞庫）
flicker（隨機字忽明忽暗）/ readjust（全體 baseAngle 隨機挪 ±0.12 再歸位）/
uneven（隨機字加 dense/faint/overlap——圓被矛盾標準弄花）/
skew（全圓 WAAPI 抽搐 skewX −16°）/ narrowWide（半圈擠半圈散 2.4s）/
eraseFew（缺口旁 3 字變「‥」墜落消散）/ pulse（radOff +12 漲縮兩下）。

# 第五章：棋盤敘事 ✅（黑白棋）

## 5.1 擬聲詞＝節拍器
- 原文的置中擬聲詞直接當 beat 標記：**啪**（stampIn 動畫 scale1.9→1 + 盤震）、
  **喀噠**（tickIn 小落）、**砰**（40px 猛震）、**喀噠喀噠——**（連鎖）。
  一拍 = 擬聲詞 + 正文 + 棋步，三者同步。

## 5.2 雙通道布局
左：正文可讀（擬聲詞維持 AO3 置中）；右：8×8 字符棋盤。
**易懂性保險**：任何時刻正文通道都是正常可讀的。

## 5.3 棋子系統
- 棋盤 = grid 字串陣列（'w/b/.'）；棋子統一「●」靠 color 分黑白
  （黑 #0c0e13 微高光、白 #e8ecf5、空「·」7% 白）；
- **翻面動畫**：`rotateY(88°) scaleY(1.06)` 150ms → 換色 → 彈回；
- **微光環棋（.ring）**：黑棋帶 `text-shadow 0 0 5px rgba(白,.5)`——
  「被塗成黑色的白棋」；點亮時 .ring-lit（金光 12px）再翻面＝身分揭露。

## 5.4 三種棋步的性格
- 蓋章（actStamp）：批次硬翻中央白棋，130ms 錯開；
- 落子（actPlace）：邊角悄悄放一子；
- 掀桌（actBang）：盤震 + 黑棋**砸進空格**（不按規則——他不下棋）+
  暗線白棋被翻黑但**留微光**（骨子裡還是白的）。

## 5.5 連鎖翻轉（跳舞）
- 距角落的棋盤距離（Chebyshev）×200ms + 隨機 120ms 錯開，一波波掃過；
- 每枚翻面彈出小「喀/噠」粒子（交替）上飄旋轉淡出；
- KEEP_BLACK 集合保留右上一撮黑——**終局是白多於黑，不是全白**；
- 結尾即時計數 `○ n ─ ● m` 淡入。

# 第六章：讀者互動機制 ✅

## 6.1 點擊推進（全系列標準）
`advance()` + busy 鎖；打字中點擊＝skipFlag 快轉；
Enter/Space 同功能；choice/按鈕 stopPropagation。

## 6.2 排版測謊（測謊排版 mockup）
- **謊言**＝該句每字 microjit（振幅僅 0.5px、0.14~0.20s 隨機）；
- **紅鯡魚**＝害怕的無辜者**整段**抖（振幅反而較大）——
  「局部的顫抖才是謊言的形狀」是機制教讀者的閱讀法；
- 指認錯誤：settleGlow 安定發光＋「這句話站得很穩」＋誤指計數；
- 指認正確：紅底線＋顫抖加劇；
- 兩謊齊 → **對質**：句子 clone 成 fixed 元素飛向畫面中央對撞、
  碎成 26 枚筆畫、真相打字浮現＋複盤課；
- **⌕ 細看**：按住把 keyframe 換成大振幅版（可近用性）；
- reduced-motion 退回波浪虛線底標示（不靠動態也能玩）。

## 6.3 選項的物理（三框 mockup 原生）
- **趨性**：pull 選項在游標 200px 內被吸向游標（k=(1−d/200)×7px）——它想被選；
- **冷靜選項**：cold-still 不動、色偏冷；
- **凋亡**：未選者 apop 動畫（scale .15 + blur）縮成一顆「·」留在原地。

## 6.4 打字測謊（輸入中⋯）⚠純本地
- 測量：首鍵延遲、總時長、Backspace/Delete 次數、
  input 事件間隔 >2s 的最長停頓、**maxDraft**（曾出現過的最長字串）；
- 猶豫判定 = `刪改≥3 ∥ 首鍵>4s ∥ 停頓>3.5s`——**完全不讀語義**
  （頁面沒有在「理解」你，只是在「看著」你——這個分寸讓它驚悚而不越界）；
- 三分支：秒回／猶豫／空白送出（「什麼都不說也是一種回答」）；
- 終幕把數據逐條還給讀者；若 maxDraft 比送出的長 → 展示「你差點送出的是——」；
- **倫理聲明必寫**：「這一切只發生在你的瀏覽器裡，哪裡也沒去」（且必須是真話）。

## 6.5 未送出的訊息（輸入中⋯）
- 「正在輸入」動點（三顆 dot 錯相跳動）；
- **幽靈草稿**：半透明斜體在動點上方逐字打出（65~135ms/字）→
  停 1.4s → 26ms/字逐字刪光 → 送出的只有「沒什麼。」
  被刪掉的才是真話；與 6.4 形成對稱結構（你看過他刪的，頁面看過你刪的）。

# 第七章：記憶與存檔 ✅

## 7.1 localStorage 標準模式 ⚠
```js
const KEY = 'mygame-save-v1';            // 帶版本號
let persistent = true;
function load(){ try{ return JSON.parse(localStorage.getItem(KEY)) ?? {}; }
                 catch{ persistent = false; return memFallback; } }
function save(m){ try{ localStorage.setItem(KEY, JSON.stringify(m)); }
                  catch{ persistent = false; memFallback = m; } }
```
- **沙盒警告**：Claude 預覽 iframe 擋 localStorage——一律 try/catch
  退回頁內記憶並在 UI 標示「沙盒模式」；下載後或 GitHub Pages 才是完全體。
- 進度收斂成**一個 state 物件**；提供「遺忘一切」重置鈕（測試必備）。

## 7.2 存檔即文字（符文）
```js
匯出: btoa(encodeURIComponent(JSON.stringify(state)))
匯入: JSON.parse(decodeURIComponent(atob(code)))
```
btoa 不吃中文所以先 encodeURIComponent。世界觀加成：存檔本身是一段文字。

## 7.3 頁面記得你（輪迴樣本三件套）
- **殘骸累積**：每次死亡 push `{t,dx,dy}` 進 MEM.residues（cap 40），
  重繪在門口——主角不記得，頁面記得；
- **跨輪解碼**：牆上刻字 decoded 每輪 +3 持久化——
  回訪的老讀者第一夜就能讀全刻字＝「頁面記得」的實感payoff；
- **來訪計數**（OC 房間）：「這是你第 N 次來看阿茸。」

# 第八章：氛圍與生態 ✅

## 8.1 粒子系統標準型
`spawn(text,x,y,{vx,vy,g,life,ground,vr,color,size})` + 主循環統一更新；
opacity 以 `life/(maxLife×0.4)` 收尾；ground:true 落地留存。
用例：雪、塵埃、♪、喀噠、血點、筆畫碎片、z。

## 8.2 人際引力（荒島）
- 引力表 `aff['a|b']`（信任正/衝突負，初始 0.3）；
- 每幀：`v += dir · aff · 0.012`；距離 <64px 個人空間斥力 0.05；
  營地向心力 0.00004；阻尼 0.93；
- 劇情拍改 aff → 讀者**看著**營地裂成兩團（社交速度的疏遠，不是跳位）；
- payoff：即時算兩派系重心、各點一堆「火」——火的位置是系統算的不是寫死的。

## 8.3 生命徵象字體（荒島）
狀態字典驅動，不做血條：

| 狀態 | 字重 | 其他 |
| ---- | ---- | ---- |
| ok | 600 | — |
| 飢餓 | 300 | — |
| 發燒 | 400 | blur 1.4px + sway 搖晃 1.6s |
| 受傷 | 500 | 紅 tint + 0.14s 微顫 |
| 虛弱 | 100 | blur .6 + opacity .55 |

名牌同步顯示狀態詞（「葉 · 發燒」）。全景一拉開誰快不行了一眼看出，零 UI。

## 8.4 桌寵 FSM（OC 房間）
- 狀態機 idle（呼吸）/walk（waypoint 表隨機挑、0.09%/幀）/bounce；
- **游標互動**：<90px 轉頭面向游標；<42px 小跳一下（冷卻 2.2s 防抽搐）；
- 點擊：蹦跳 3 循環 + 隨機台詞（OC_LINES 表）；
- 生態細節：毛球每 2.6s 冒「z」、被點會醒（catAwake 旗標停 z）；
  塵埃 0.7s 一粒緩緩上飄；燈的光暈 5s 呼吸。
- **可探索房間**：FURNITURE 表 `{glyph,name,x%,y%,size,vignette}`，
  點任何家具打出小故事（typeToken 打斷舊打字）。

# 第九章：設計紀律總表

1. **主題決定物理**——每部作品鎖定一個招牌機制，不把特效全倒進去。
2. **雙通道易懂性**——物理演出再狂，永遠留一條正常可讀的通道
   （圓心字幕／左欄正文／中框旁白）。
3. **一拍一事件**——框變化、布局大招只在拍與拍之間；分裂/易位整場限一兩次。
4. **逐字隨機**——振幅/頻率/延遲逐字隨機才像活的。
5. **速度即演技**——walk speed、打字語速、停頓長度都是台詞。
6. **對照組是演出的一部分**——不被影響的元素（SAFE/SILENT/紅鯡魚）證明系統有規則。
7. **擦除≠彈射**——被抹去的消失，被拋棄的留下屍體；殘骸是敘事資產。
8. **世界與劇情連動**——雪停、火點、標題變，一個旗標的事，收穫不成比例。
9. **cleanup 強制**——每個效果可撤銷，切換零殘留。
10. **prefers-reduced-motion 全支援**——動態機制準備非動態替代
    （測謊→波浪底線；倒帶→淡出）。
11. **偷看讀者要有分寸**——只量行為不讀語義；倫理聲明必寫且必須是真話。
12. **localStorage 一律 try/catch**——沙盒退回頁內記憶並標示。
13. **資料表驅動**——正式開工＝換資料表，不是從零寫程式。
